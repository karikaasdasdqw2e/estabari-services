# تفعيل لوحة تحكم مجمع خدمات إصطباري

لوحة التحكم موجودة على:

```text
/admin.html
```

مثال على GitHub Pages:

```text
https://karikaasdasdqw2e.github.io/estabari-services/admin.html
```

## لماذا كلمة المرور ليست داخل GitHub؟

ملفات الموقع عامة ويمكن لأي شخص قراءة كود JavaScript. لذلك لا يتم تخزين كلمة مرور الإدارة داخل `admin.js` أو `config.js`، بل يتم التحقق منها من خلال Supabase Auth.

## خطوات التفعيل مرة واحدة

1. افتح مشروع Supabase.
2. افتح `SQL Editor` وشغّل ملف `supabase-schema.sql` المحدّث كاملًا.
3. افتح `Authentication` ثم `Users`.
4. اضغط `Add user` أو `Create new user`.
5. استخدم البريد الداخلي التالي:

```text
karika@estabari.local
```

6. ضع كلمة المرور التي اخترتها للوحة التحكم.
7. فعّل خيار `Auto Confirm User` أو اجعل المستخدم `Confirmed`.
8. من إعدادات Auth عطّل التسجيل العام `Allow new users to sign up` لأن الموقع لا يحتاج حسابات عامة.

بعد ذلك افتح `admin.html` وسجّل الدخول باستخدام:

```text
اسم المستخدم: karika
كلمة المرور: كلمة المرور التي وضعتها داخل Supabase
```

## إمكانيات لوحة التحكم

- عرض إحصائيات الخدمات والزوار.
- إضافة خدمة جديدة.
- تعديل جميع بيانات أي خدمة.
- نشر أو إيقاف الخدمة.
- رفض الخدمة أو إبقاؤها قيد المراجعة.
- توثيق الخدمة.
- جعل الخدمة مميزة لتظهر أولًا.
- حذف الخدمة نهائيًا.
- البحث والتصفية حسب القسم والحالة.
- ترتيب الخدمات.
- تصدير جميع الخدمات إلى ملف CSV.
- تسجيل خروج آمن مع جلسة Supabase Auth.

## الحماية

سياسات Row Level Security تمنع أي مستخدم غير حساب الإدارة من تعديل أو حذف الخدمات، حتى لو حاول إرسال طلبات مباشرة إلى Supabase.
