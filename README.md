# مجمع خدمات إصطباري

منصة عربية متجاوبة تساعد أهالي قرية إصطباري في الوصول إلى مقدمي الخدمات المحليين بسهولة، وتتيح لمقدم الخدمة تسجيل بياناته وموقعه للتواصل المباشر عبر الهاتف أو واتساب.

## المزايا الحالية

- واجهة عربية RTL متوافقة مع الهاتف والكمبيوتر.
- بحث بالاسم أو نوع الخدمة أو وصفها أو المنطقة.
- تصفح الخدمات من خلال أقسام واضحة.
- تسجيل مقدم خدمة مع رقم الهاتف، واتساب، العنوان، المواعيد وتحديد الموقع.
- أزرار اتصال وواتساب وفتح الاتجاهات على الخريطة.
- حفظ الخدمات المفضلة على نفس الجهاز.
- دعم Supabase اختياري لحفظ البيانات ومراجعتها مركزيًا.
- وضع محلي تلقائي للتجربة قبل ربط قاعدة البيانات.
- تطبيق ويب قابل للتثبيت PWA.

## تشغيل المشروع محليًا

يمكن فتح `index.html` مباشرة، أو تشغيل خادم محلي:

```bash
python -m http.server 8080
```

ثم افتح `http://localhost:8080`.

## ربط قاعدة البيانات Supabase

1. أنشئ مشروعًا جديدًا في Supabase.
2. افتح SQL Editor وشغّل محتوى الملف `supabase-schema.sql`.
3. انسخ Project URL وAnon Key.
4. ضعهما داخل `config.js`.
5. الطلبات الجديدة تُحفظ بالحالة `pending`، ويمكن اعتمادها من لوحة Supabase بتغيير `status` إلى `approved`.

> مفتاح Supabase العام Anon Key مصمم للاستخدام داخل الواجهة، ولا يجب وضع Service Role Key في الموقع مطلقًا.

## النشر على GitHub Pages

يوجد Workflow جاهز داخل `.github/workflows/deploy-pages.yml`.

من إعدادات الريبو افتح:

`Settings → Pages → Build and deployment → Source: GitHub Actions`

بعدها سيتم نشر أي تحديث على فرع `main` تلقائيًا.

## الملفات الأساسية

- `index.html`: هيكل الموقع والواجهات.
- `styles.css`: التصميم المتجاوب.
- `app.js`: البحث والتصنيفات والتسجيل والتواصل.
- `config.js`: إعدادات Supabase وبيانات الإدارة.
- `supabase-schema.sql`: قاعدة البيانات وسياسات الأمان.
- `manifest.json` و`sw.js`: دعم تثبيت الموقع والعمل الأساسي دون إنترنت.

## المرحلة التالية المقترحة

- لوحة إدارة لاعتماد أو رفض الخدمات.
- تسجيل دخول مقدم الخدمة وتعديل بياناته.
- تقييمات وتعليقات موثقة.
- إعلانات محلية مدفوعة وخدمات مميزة.
- تنبيهات عند إضافة خدمة جديدة.
- صفحة مستقلة لكل مقدم خدمة وتحسين الظهور في محركات البحث.
